<task type="auto">
  <name>Task 2: Define shared TypeScript types</name>
  <files>src/lib/types/tool-result.ts, src/lib/types/weather.ts, src/lib/types/buoy.ts, src/lib/types/location.ts</files>
  <action>
  Create the shared type definitions used by all MCP tool servers.

  **tool-result.ts** вЂ” Generic tool result wrapper:
  ```typescript
  export type ToolResult<T> =
    | { success: true; data: T }
    | { success: false; error: string; source: string };
  ```

  **weather.ts** вЂ” Weather data types:
  - `CurrentWeather`: temperature, feelsLike, humidity, windSpeed, windDirection, pressure, uvIndex, visibility, weatherCode, description, timestamp (all numbers/strings, units in metric)
  - `HourlyForecast`: array of { time, temperature, precipitation, windSpeed, windDirection, weatherCode }
  - `DailyForecast`: array of { date, tempMax, tempMin, precipitationSum, weatherCode, description }
  - `MarineWeather`: seaSurfaceTemp, waveHeight, wavePeriod, waveDirection, swellHeight, swellPeriod, swellDirection, windWaveHeight, windWavePeriod

  **buoy.ts** вЂ” Buoy data types:
  - `BuoyStation`: id, name, lat, lon, type ('buoy' | 'land' | 'ship'), owner (optional)
  - `BuoyObservation`: stationId, timestamp, waveHeight, wavePeriod (DPD), windSpeed, windDirection, waterTemp, airTemp, pressure, dewpoint, visibility вЂ” all `number | null` (null for missing MM values)
  - `BuoyHistory`: stationId, observations: BuoyObservation[] (last 24h)

  **location.ts** вЂ” Location types:
  - `Coordinates`: lat, lon
  - `GeocodingResult`: placeName, coordinates: Coordinates, relevance: number, context (city, region, country)
  </action>
  <verify>npx tsc --noEmit passes with all type files</verify>
  <done>All types exported, no TypeScript errors, types accurately model the domain data</done>
</task>
