<task type="auto">
  <name>Task 1: Implement weather tool functions</name>
  <files>src/lib/tools/weather.ts</files>
  <action>
  Create three tool functions that call the Open-Meteo API server-side and return typed results.

  **getCurrentWeather(lat: number, lon: number): Promise<ToolResult<CurrentWeather>>**
  - Fetch: `https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}&current=temperature_2m,relative_humidity_2m,apparent_temperature,precipitation,weather_code,wind_speed_10m,wind_direction_10m,surface_pressure,uv_index,visibility&timezone=auto`
  - Map response to CurrentWeather type
  - Include a human-readable `description` derived from WMO weather code (clear, cloudy, rain, etc.)
  - Return ToolResult wrapper — success with data or error with source "open-meteo"
  - NEVER throw — always return structured error

  **getForecast(lat: number, lon: number, type: 'hourly' | 'daily'): Promise<ToolResult<HourlyForecast[] | DailyForecast[]>>**
  - If type='hourly': fetch with `&hourly=temperature_2m,precipitation_probability,precipitation,wind_speed_10m,wind_direction_10m,weather_code&forecast_hours=48`
  - If type='daily': fetch with `&daily=temperature_2m_max,temperature_2m_min,precipitation_sum,weather_code&forecast_days=7`
  - Map to appropriate type array
  - Include timezone info from response

  **getMarineWeather(lat: number, lon: number): Promise<ToolResult<MarineWeather>>**
  - Fetch: `https://marine-api.open-meteo.com/v1/marine?latitude={lat}&longitude={lon}&current=sea_surface_temperature,wave_height,wave_period,wave_direction,swell_wave_height,swell_wave_period,swell_wave_direction,wind_wave_height,wind_wave_period`
  - Map to MarineWeather type
  - Handle gracefully when location is inland (Open-Meteo returns null/empty for non-marine locations)

  Create a helper function `weatherCodeToDescription(code: number): string` that maps WMO weather codes to human-readable descriptions (0=Clear sky, 1-3=Partly cloudy, 45-48=Fog, 51-55=Drizzle, 61-65=Rain, 71-75=Snow, 80-82=Showers, 95-99=Thunderstorm).
  </action>
  <verify>Call each function with Portland, OR coordinates (45.5, -122.6) and verify structured response</verify>
  <done>All three functions return valid ToolResult with typed data for known coordinates</done>
</task>

<task type="auto">
  <name>Task 2: Create weather API route</name>
  <files>src/app/api/weather/route.ts</files>
  <action>
  Create a Next.js App Router API route that exposes weather tools via GET requests.

  GET /api/weather?lat={lat}&lon={lon}&tool={tool}&type={type}

  Where `tool` is one of: `current`, `forecast`, `marine`
  And `type` is optional, used only for forecast: `hourly` or `daily`

  Validate query parameters:
  - lat and lon are required numbers between -90/90 and -180/180
  - tool is required and must be one of the valid values
  - Return 400 with descriptive error for invalid params

  Call the appropriate tool function and return the ToolResult as JSON.
  Set Cache-Control header: `public, s-maxage=300` (5 min server cache).
  </action>
  <verify>curl "http://localhost:3000/api/weather?lat=45.5&lon=-122.6&tool=current" returns valid JSON</verify>
  <done>API route returns weather data for all three tool types, validates params, sets cache headers</done>
