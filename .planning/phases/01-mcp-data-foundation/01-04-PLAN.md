<task type="auto">
  <name>Task 1: Implement geocoding tool function</name>
  <files>src/lib/tools/location.ts</files>
  <action>
  Create a geocoding tool function that resolves location queries using the Mapbox Geocoding API.

  **resolveLocation(query: string): Promise<ToolResult<GeocodingResult[]>>**
  - Fetch: `https://api.mapbox.com/geocoding/v5/mapbox.places/${encodeURIComponent(query)}.json?access_token=${token}&limit=5`
  - Read token from `process.env.NEXT_PUBLIC_MAPBOX_TOKEN`
  - Map response features to GeocodingResult type:
    - `placeName` from `feature.place_name`
    - `coordinates` from `feature.center` (note: Mapbox returns [lon, lat], flip to {lat, lon})
    - `relevance` from `feature.relevance`
    - `context` object with city, region, country extracted from `feature.context` array (each context item has an `id` like "place.123", "region.456", "country.789" — use the id prefix to determine type)
  - Handle missing token: return error with message "Mapbox token not configured"
  - Handle empty results: return success with empty array
  - Handle API errors: return structured error with source "mapbox"
  - NEVER throw — always return structured error

  Supports these query types (Mapbox handles all natively):
  - City names: "Portland, OR"
  - Zip codes: "97201"
  - Coordinate strings: "-122.6, 45.5"
  - Addresses: "123 Main St, Portland"
  </action>
  <verify>Call resolveLocation("Portland, OR") and verify it returns coordinates near (45.5, -122.6)</verify>
  <done>resolveLocation returns valid GeocodingResult array for city names, zip codes, and coordinates</done>
</task>
