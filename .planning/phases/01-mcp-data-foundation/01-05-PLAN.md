<task type="auto">
  <name>Task 1: Implement in-memory cache with TTL</name>
  <files>src/lib/cache.ts</files>
  <action>
  Create a caching utility with TTL expiry and in-flight request deduplication.

  **Data structures:**
  - `cache: Map<string, { data: unknown, expiry: number }>` — cached responses
  - `inflight: Map<string, Promise<unknown>>` — in-flight request promises for deduplication

  **cachedFetch<T>(key: string, fetcher: () => Promise<T>, ttlMs?: number): Promise<T>**
  - Default TTL: 300_000 (5 minutes)
  - Check cache first: if key exists and not expired, return cached data
  - Check inflight: if same key is already being fetched, return the existing promise (deduplication)
  - Otherwise: call fetcher(), store promise in inflight map, on resolve store result in cache with expiry, remove from inflight
  - On fetcher error: remove from inflight, do NOT cache the error, re-throw (let the tool function handle it)
  - Expired entries are lazily cleaned up on access (no background timer needed)

  **clearCache(keyPrefix?: string): void**
  - If keyPrefix provided, clear all entries whose key starts with prefix
  - If no prefix, clear entire cache
  - Useful for testing and manual invalidation

  Export both functions as named exports.
  </action>
  <verify>Call cachedFetch twice with same key — second call should return instantly without calling fetcher</verify>
  <done>Cache returns cached data within TTL, deduplicates concurrent requests, does not cache errors</done>
</task>

  **Buoy tools** (src/lib/tools/buoys.ts):
  - Wrap getBuoyObservations fetch with key: `buoy:observations:${stationId}`
  - Wrap getBuoyHistory fetch with key: `buoy:history:${stationId}`
  - getNearbyBuoys does NOT need caching (pure computation, no API call)
  - TTL: 300_000 (5 minutes, default)

  **Location tools** (src/lib/tools/location.ts):
  - Wrap resolveLocation fetch with key: `location:resolve:${query}`
  - TTL: 86_400_000 (24 hours — location data is very stable)

  The cache wraps the raw fetch, NOT the entire tool function. The ToolResult error handling remains outside the cache so errors are not cached.
