  <name>Task 1: Implement Haversine distance utility</name>
  <files>src/lib/utils/distance.ts</files>
  <action>
  Create a utility function that calculates great-circle distance between two points using the Haversine formula.

  **haversineDistance(lat1: number, lon1: number, lat2: number, lon2: number): number**
  - Returns distance in **miles** (not km)
  - Uses Earth radius = 3958.8 miles
  - Formula: `a = sin²(Δlat/2) + cos(lat1) * cos(lat2) * sin²(Δlon/2)`, `c = 2 * atan2(√a, √(1−a))`, `d = R * c`
  - Convert degrees to radians internally
  - This is critical for accurate ocean proximity — do NOT use simple lat/lon subtraction

  Export the function as a named export.
  </action>
  <verify>haversineDistance(45.5, -122.6, 46.163, -124.487) should return ~95-100 miles (Portland to Columbia River Bar buoy)</verify>
  <done>Haversine function returns accurate great-circle distances in miles</done>
</task>

<task type="auto">
  <name>Task 2: Implement NDBC text parser and buoy tool functions</name>
  <files>src/lib/tools/buoys.ts</files>
  <action>
  Create three tool functions for NOAA buoy data, plus an internal NDBC text parser.

  **Internal: parseNDBCText(text: string): BuoyObservation[]**
  - Parse fixed-width text format from NDBC realtime2 endpoint
  - First line starts with `#YY` — column names
  - Second line starts with `#yr` — units (skip this)
  - Remaining lines are data rows, space-separated
  - Column mapping:
    - YY MM DD hh mm → timestamp (construct Date from UTC components)
    - WDIR → windDirection (degrees)
    - WSPD → windSpeed (m/s)
    - GST → gustSpeed (m/s)
    - WVHT → waveHeight (meters)
    - DPD → wavePeriod (seconds, dominant)
    - APD → avgWavePeriod (seconds)
    - MWD → meanWaveDirection (degrees)
    - PRES → pressure (hPa)
    - ATMP → airTemp (°C)
    - WTMP → waterTemp (°C)
    - DEWP → dewpoint (°C)
    - VIS → visibility (nautical miles)
  - Missing data handling: `MM`, `99.0`, `999.0`, `9999.0` → null
  - Return array of BuoyObservation objects, newest first

  **getNearbyBuoys(lat: number, lon: number, radiusMiles?: number, limit?: number): Promise<ToolResult<(BuoyStation & { distanceMiles: number })[]>>**
  - Default radiusMiles = 100, default limit = 10
  - Load static station list from `src/data/buoy-stations.json`
  - Calculate haversineDistance from (lat, lon) to each station
  - Filter stations within radiusMiles
  - Sort by distance ascending
  - Limit to `limit` results
  - Return stations with `distanceMiles` appended to each
  - This is a pure computation — no API call needed

  **getBuoyObservations(stationId: string): Promise<ToolResult<BuoyObservation>>**
  - Fetch: `https://www.ndbc.noaa.gov/data/realtime2/{stationId}.txt`
  - Parse with parseNDBCText
  - Return the FIRST (most recent) observation
  - Handle 404 (station not found) and network errors gracefully
  - NEVER throw — return structured error with source "ndbc"

  **getBuoyHistory(stationId: string): Promise<ToolResult<BuoyObservation[]>>**
  - Fetch same URL as getBuoyObservations
  - Parse with parseNDBCText
  - Filter to observations within the last 24 hours
  - Return the filtered array (could be 24-48 entries depending on reporting frequency)
  - Handle errors same as getBuoyObservations
  </action>
  <verify>Call getBuoyObservations("46029") for Columbia River Bar and verify structured response with wave/wind data</verify>
  <done>All three buoy functions return valid ToolResult, NDBC text parsed correctly with null for missing values</done>
</task>
